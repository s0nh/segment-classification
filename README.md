# segment-classification

## 일정

* 1 주차 (2025.11.6~2025.11.13)

  __각자 맡은 데이터 전처리__
  ```
  * 왕민 1, 4, 5 (회원 정보, 청구입금 정보, 잔액 정보)
  * 홍정표 6, 7, 8 (채널 정보, 마케팅 정보, 성과 정보)
  * 손혜성 2, 3 (신용 정보, 승인매출 정보)
  ```
  결측치 존재하는 독립변수 총 31개 처리
  
  ### 범주형 데이터
  트리기반 모델을 사용하여 모델링할 계획이기 때문에 원핫인코딩은 적용하지 않고 레이블 인코딩만 적용할 예정임
  1. 결측치를 `None`(문자열)으로 대체 또는 과반수를 차지하는 클래스가 있다면 그 클래스에 병합
  2. 모든 경우의 수에 대해 카이제곱 검정 → 통계량이 높은 케이스를 채택하여 결측치 채움
  

  ### 연속형 데이터

  1. 좌우 대칭이면 `mean` 비대칭이면 `median` 으로 대체
  2. 기존에 결측치였다는 사실을 나타내는 이진데이터를 추가함 
  3. 이진데이터에 대해서 카이제곱 검정 → 유의미한 결과가 나온다면 채택
